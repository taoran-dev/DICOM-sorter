Help & About

Instructions for Use (IFU)

Purpose: Client-side tool to sort and optionally anonymize DICOM studies. Output is organized by day (D0, D1, ‚Ä¶) and clusters using FrameOfReferenceUID (T1, T2, ‚Ä¶) so CT and related RS/RP/RD import together and remain auto-registered.

Intended users: Research and clinical engineering staff familiar with DICOM handling. Not a medical device.

Supported browsers: Latest Chrome/Edge recommended. ‚ÄúSave to Folder‚Äù requires the File System Access API (Chromium-based).

Workflow

  1. Open this HTML file in a supported browser.
  2. Click ‚ÄúSelect Directory‚Äù or drop a folder of DICOMs into the drop zone.
  3. Review the Planned Output (Preview) on the left. It shows D/T folders and series with counts; RS/RP/RD display their identifiers.
  4. (Optional) Expand Anonymization (optional) and enable anonymization; configure patient fields if needed.
  5. Save: Choose ‚ÄúSave To Folder‚Äù (fast, streams individual files) or ‚ÄúDownload Processed Files‚Äù (ZIP with progress).
  6. Verify images for visible PHI (burned-in text/overlays) before sharing.

Output structure

  • Folders: D0/, D1/, ‚Ä¶ by acquisition date; inside each, T1-CT-RS/, T2-CT-RP-RD/, etc. by shared FrameOfReferenceUID.
  • Series subfolders: SERIES_{SeriesNumber}_{Modality}/
  • Filenames:
    • RP(PLAN)_{PlanID}_{YYYYMMDD}_{HHMMSS}_{Seq}.dcm
    • RS(STRUCT)_{StructID}_{YYYYMMDD}_{HHMMSS}_{Seq}.dcm
    • RD(DOSE)_{AssocPlanID}_{YYYYMMDD}_{HHMMSS}_{Seq}.dcm
    • {MOD}_{YYYYMMDD}_{HHMMSS}_{Seq}.dcm for other modalities

Anonymization summary

  • Removes PHI from standard header tags aligned with DICOM PS3.15 Basic Application Level Confidentiality (header‚Äëonly subset); retains Type 2 elements as empty.
  • Remaps Study/Series/SOP/FrameOfReference and referenced UIDs consistently; preserves SOP Class/Transfer Syntax UIDs.
  • Optional: strip Private tags (odd groups).
  • Limitation: Pixel data not analyzed; burned-in PHI is not removed. Manual verification required.

Troubleshooting

  • Large studies: prefer ‚ÄúSave To Folder‚Äù to avoid huge ZIP buffers. Progress appears during save.
  • Memory error (‚ÄúArray buffer allocation failed‚Äù): use streaming save paths as above.
  • ‚ÄúSave To Folder‚Äù missing: use Chrome/Edge (File System Access API required).
  • No preview after selecting: ensure a directory was chosen and files are readable.
